# Конституционный ассистент — пилотная версия

Публичный чат-бот для граждан («Constitutional Assistant» из раздела 2.2 концепции).
Помогает в диалоге собрать данные, нужные для подачи обращения в Конституционный Суд РК
по статьям 44–45 Конституционного закона, и показывает прогресс сбора данных в виде
живой «Справки обращения» рядом с чатом.

Стек повторяет архитектуру DermAI: статичный фронтенд (HTML/CSS/JS) + Netlify Function,
которая проксирует запросы к Gemini API и не раскрывает ключ в браузере.

## Структура проекта

```
constitutional-assistant/
├── index.html                  # разметка страницы
├── styles.css                  # стили
├── script.js                   # логика чата на клиенте
├── netlify/functions/chat.js   # серверная функция (вызов Gemini API)
├── netlify.toml                # конфигурация Netlify
└── package.json
```

## Деплой на Netlify

1. **Получите ключ Gemini API** в Google AI Studio: https://aistudio.google.com/apikey
2. Загрузите эту папку в Netlify — самый быстрый способ: зайти в Netlify → "Add new site" →
   "Deploy manually" и перетащить папку (или подключить Git-репозиторий с этой папкой).
3. В **Site settings → Environment variables** добавьте переменную:
   ```
   GEMINI_API_KEY = ваш_ключ
   ```
4. Запустите деплой (Trigger deploy / Deploy site). Netlify автоматически подхватит
   функцию из `netlify/functions/chat.js` благодаря `netlify.toml`.
5. Откройте сайт — чат сразу должен отвечать через Gemini.

## Локальная проверка (опционально)

Если установлен Netlify CLI:

```bash
npm install -g netlify-cli
netlify dev
```

Команда поднимет и статику, и функцию локально с переменными окружения из `.env`
(создайте файл `.env` с `GEMINI_API_KEY=...` рядом с проектом — он не должен попадать в Git).

## Что это демонстрирует, а что — нет (важно для презентации пилота)

✅ Реализовано в этой версии:
- Проверка подсудности первым шагом диалога
- Пошаговый сбор полей по ст. 44–45 с живой визуализацией прогресса
- Объяснение терминов по запросу
- Генерация итогового черновика обращения текстом в конце диалога
- Ключ API скрыт на сервере (Netlify Function), не передаётся в браузер

⚠️ Упрощения пилота (стоит явно проговаривать инвесторам/комитету):
- Вместо полноценной RAG-базы (Конституция + НПА + прецеденты, как описано в разделе 2.3
  концепции) используется единый системный промпт с кратким изложением требований ст. 44–45.
  Для продакшна нужна реальная база знаний и retrieval, иначе модель может ошибаться в деталях.
- Нет on-premises / закрытого контура — Gemini API облачный. Для реального проекта,
  согласно концепции, потребуется локально развёрнутая open-source модель (LLaMA/Mistral/Gemma).
- Нет сохранения данных (Supabase) и аутентификации — каждая сессия живёт только в памяти браузера.
- Нет автоматической интеграции с базой НПА (Adilet/Paragraph) для подстановки реквизитов акта.

Эти ограничения соответствуют масштабу «пилота для демонстрации концепции», а не боевой
системы — именно так стоит представлять сайт на защите/презентации.

## Смена модели

Модель задаётся константой `MODEL` в `netlify/functions/chat.js` (по умолчанию
`gemini-2.5-flash`). Можно заменить на любую доступную по вашему ключу модель Gemini.
